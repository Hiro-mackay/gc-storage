## 2. 短期計画（3-6ヶ月）

### v1.0 - コアMVP ✅ In Progress

**目標**: 基本的なファイルストレージ機能のリリース

| 機能 | 状態 | 説明 |
|------|------|------|
| ファイルアップロード/ダウンロード | ✅ 完了 | マルチパートアップロード対応 |
| フォルダ管理 | ✅ 完了 | 作成、移動、削除、名前変更 |
| ユーザー認証 | ✅ 完了 | JWT + OAuth (Google, GitHub) |
| 権限管理 | ✅ 完了 | PBAC + ReBAC |
| 共有リンク | ✅ 完了 | 期限付き、パスワード保護 |
| グループ機能 | 🔄 進行中 | グループ作成、メンバー管理 |
| 検索 | 🔄 進行中 | ファイル名検索、フィルタリング |

### v1.1 - UX改善 + 追加機能

**目標**: ユーザー体験の向上と基本機能の拡充

| 機能 | 優先度 | 説明 |
|------|--------|------|
| ゴミ箱機能 | 高 | 論理削除、30日後に自動物理削除 |
| お気に入り | 高 | ファイル/フォルダのブックマーク |
| 最近使ったファイル | 高 | アクセス履歴の表示 |
| ドラッグ&ドロップ改善 | 高 | ファイル移動のUI改善 |
| プレビュー拡張 | 中 | テキストファイル、コード表示 |
| タグ機能 | 中 | カスタムタグによる整理 |
| 一括操作 | 中 | 複数ファイルの選択・操作 |

## 5. 技術的ロードマップ

### インフラストラクチャ

| フェーズ | 項目 | 説明 |
|---------|------|------|
| Phase 1 | マルチリージョン対応 | 災害復旧、レイテンシ改善 |
| Phase 2 | CDN 統合 | 静的アセット、ダウンロード高速化 |
| Phase 3 | エッジコンピューティング | プレビュー生成の分散処理 |

### パフォーマンス

| 目標 | 現状 | 目標値 |
|------|------|--------|
| API レイテンシ (P95) | 500ms | < 200ms |
| ファイル一覧表示 | 1s | < 500ms |
| 検索レスポンス | 2s | < 1s |
| アップロードスループット | 50MB/s | 100MB/s |

### セキュリティ

| フェーズ | 項目 |
|---------|------|
| 継続 | 脆弱性スキャン自動化 |
| 継続 | ペネトレーションテスト（年次） |
| v2.0 | エンドツーエンド暗号化（オプション） |
| v2.0 | ゼロ知識暗号化（検討） |
